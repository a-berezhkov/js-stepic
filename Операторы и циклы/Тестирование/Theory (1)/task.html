Фреймворк для тестирования mocha

Информация и хороший мануал можно найти на официальном сайте - https://mochajs.org.

Установка фреймворка производится с помощью менеджера зависимостей npm.

$ npm install --global mocha

После установки фреймворка нужно создать в папке проекта файл test.js и внести туда следующий код.

var assert = require('assert');
describe('Array', function () {
  describe('#indexOf()', function () {
    it('should return -1 when the value is not present', function () {
      assert.equal([1, 2, 3].indexOf(4), -1);
    });
  });
});

Для запуска теста достаточно в консоли из директории проекта выполнить команду mocha.

После успешного запуска, давайте разберемся, что же произошло.

describe('Array', function () {
  
});

Методы describe представляет собой описание функциональности, которую мы проверяем.  

То же самое относится и к этому участку. 

describe('#indexOf()', function () {
     
});

Если посмотреть на вывод результат выполнения теста - там будет как подпись "Array" так и "#indexOf()". 

describe предназначен только для группировок самих тестов -  блоков it. Если говорить об общем синтаксисе, то он выглядит следующим образом describe("заголовок", function() { ... }).

Блоки it служат для описания самих тестов. Самое важное - то бы выполнение результата было понятным.

it('should return -1 when the value is not present', function () {
    assert.equal([1, 2, 3].indexOf(4), -1);
});

В примере выводится сообщение, что тест "должен вернуть -1 когда значение отсутсвует". 

Функция assert.equal()имеет два аргумента - первый условие которое проверяется, в нашем случае [1, 2, 3].indexOf(4) , второе - ожидаемый результат -1. Если assert.equal() вернет true - тест будет считаться успешно пройденным.
