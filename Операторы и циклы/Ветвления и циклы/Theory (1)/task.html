while/do... while

Общий синтаксис 

while (condition) {
  // код (тело цикла)
}

Можно представить в виде

пока (условие истино ( т.е. true)) {
   что-то делать
}

Можно сделать цикл, который будет бесконечно выводить "Привет" (не стоит повторять).

while (1) {    
    console.log("Привет");
}
// или так
while (true) {
    console.log("Привет");
}

Циклы нужны для того что бы повторять какое-то действие неограниченное (чаще просто заранее неизвестное) число раз. Как и во всех курсах, вот пример бесполезного вывода от 1 до 10. Это славная традиция, не будем ее нарушать, только чуть дополним, для понимания.

let i = 1;
while (i <= 10) { 
  console.log( i );
  i++;
}

// 1я итерация (проход цикла)

если i <= 10 {       // i = 1
    вывести(i);      // i = 1
    увеличить i на 1 // i = 2
}

// 2я итерация (проход цикла)

если i <= 10 {       // i = 2
    вывести(i);      // i = 2
    увеличить i на 1 // i = 3
}

// .....

// 10я итерация (проход цикла)

если i <= 10 {       // i = 10
    вывести(i);      // i = 10
    увеличить i на 1 // i = 11
}

Как видно, на 10й итерации (итерацией называется один проход цикла) переменной i присваивается значение 11, выражение i<=10 становится false и цикл дальше не выполняется. 

Код выше можно переписать и в одну строчку:

let i = 1;
while (i <= 10) console.log( i++ );

Можно циклы использовать для работы со строками. Код ниже выводит каждый второй символ.

let text = "Привет,JS";
let index = 0;
while (index < text.length){
    console.log(text[index]);
    index+=2;
}

Можно ждать ответа от пользователя. Пока пользователь не введет "4", цикл будет повторяться. 

answer = prompt("Сколько будет 2 + 2")
while (answer != 4){
    alert("Неправильно")
    answer = prompt("Сколько будет 2 + 2") 
}

 В примере выше все хорошо, кроме того, что есть две одинаковые строчки. А дублирование кода - плохая идея. Когда у вас есть повторяющиеся участки, скорее всего вы делаете что-то не так. Пример выше можно исправить, используя do...while. Пишется он так.

do {
  // тело цикла
} while (condition);

Идея такая же как и у while, разница лишь в том, что сначала выполнится do {}, а потом проверка условия. 

Теперь можно переписать код с вопросом более лаконично. Однако придется пожертвовать сообщением об ошибки, так как условия еще не проходили, а без них тут не обойтись. 

do {
    answer = prompt("Сколько будет 2 + 2")
} while(answer != 4)

Вот и все, разобрали первый цикл (даже почти два.... хотя один... эх... полтора?!)
