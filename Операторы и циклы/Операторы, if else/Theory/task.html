Операторы сравнения

Казалось бы с булевыми переменными все просто -  имеет только два значения - true и false. Обычно переменные, которые имеют такой тип начинают со слова is ( isDone) или bool (boolDone). 

let workIsDone = true; // да, работа сделана
let workIsDone = false; // нет, работа не сделана

Этот типа важен для различных условий. Давайте сравним числа 3 и 4.

console.log(3>4); // false
console.log(3<4); // true

Так же можно проверить равенство (==) или неравеснство (!=) значений.

console.log(3==4); // false
console.log(3!=4); // true

Есть еще строгое равенство (===), которое проверяет не только равенство значения, но и типа.

console.log(3===3); // true
console.log(3==="3"); // false

 Так же как и математике можем использовать сравнени больше или равно - >= и меньше или равно <=.

console.log(3>=3); // true
console.log(3<=3); // true

Операции сравнения над строками 

Строки так же можно сравнивать, но надо понимать, что они строки сравниваются посимвольно.

alert( 'Б' > 'А' ); // true
alert( 'Коты' > 'Котя' ); // false
alert( 'Сонный' > 'Сон' ); // true

Алгоритм сравнения двух строк довольно прост:

  1. Сначала сравниваются первые символы строк.
  2. Если первый символ первой строки больше (меньше), чем первый символ второй, то первая строка больше (меньше) второй. Сравнение завершено.
  3. Если первые символы равны, то таким же образом сравниваются уже вторые символы строк.
  4. Сравнение продолжается, пока не закончится одна из строк.
  5. Если обе строки заканчиваются одновременно, то они равны. Иначе, большей считается более длинная строка.

Приведённый выше алгоритм сравнения похож на алгоритм, используемый в словарях и телефонных книгах, но между ними есть и различия.

Например, в JavaScript имеет значение регистр символов. Заглавная буква "A" не равна строчной "a". Какая же из них больше? Строчная "a". Почему? Потому что строчные буквы имеют больший код во внутренней таблице кодирования, которую использует JavaScript (Unicode).

Сравнение разных типов

Строка и число

При сравнении значений разных типов JavaScript приводит каждое из них к числу.

console.log( '2' > 1 ); // true, строка '2' становится числом 2
console.log( '01' == 1 ); // true, строка '01' становится числом 1

Булевый тип и число

Булевый тип будет преобразован в 1 и 0

console.log( true == 1 ); // true
console.log( false == 0 ); // true

Сравнение с null и undefined 

Тут все гораздо интересней. 

console.log( null === undefined ); // false

Это логично, так как null и undefined имеют разные типы.

console.log( null == undefined ); // true

А вот строчка выше не очень логиячна, поэтому ее надо просто запомнить.

Эти значения равны друг другу и не равны никаким другим значениям. Это специальное правило языка. 

При использовании математических операторов и других операторов сравнения < > <= >=

Значения null/undefined преобразуются к числам: null становится 0, а undefined – NaN.

Сравнение 0 и null

console.log( null > 0 );  // false
console.log( null == 0 ); // false
console.log( null >= 0 ); // true

Сравнения преобразуют null в число, рассматривая его как 0. Поэтому первый пример null >= 0 имеет false.

То, что вот второй строке null == 0  возвраващает false логично, об этом писали выше)

И исходя того, что null бужет преобразован к 0 - третья строка так же имеет место быть. 

Значение undefined несравнимо с другими значениями

console.log( undefined > 0 );  // false
console.log( undefined < 0 );  // false
console.log( undefined == 0 );  // false
console.log( undefined === 0 );  // false
console.log( undefined > 1 );  // false
console.log( undefined < 1 );  // false
console.log( undefined  == 1 );  // false
console.log( undefined  === 1 );  // false
console.log( undefined  == null );  // true
console.log( undefined  === null );  // false
console.log( undefined  < null );  // false
console.log( undefined  > null );  // false
console.log( undefined  == NaN );  // false
console.log( undefined  === NaN );  // false

В общем, надо стараться избегать сравнения с undefined/null.
