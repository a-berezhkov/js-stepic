Имена переменных

Кажется этот раздел не важен. Поэтому вот пара переменных и попробуйте представить, что они значат.

let a = 120;
const i = 3.14;
let we = 70;
let g = "Привет";
let o = 0;

Если вы не поняли, то ничего страшного.

Представьте, что код состоит из таких переменных. Ужас, да и только.

А теперь перепишем этот код, что бы стало понятно.

let personHeight = 120;
const piNumber = 3.14;
let personwWight = 70;
let welcomeMessage = "Привет";
let isAgree = 0;

Гораздо лучше. Сразу стало понятно, что и где хранится. Возможно isAgree вносит некоторую смуту, но забегая вперед, это вариант булевой переменной, которая принимает только два значения: true ("правда") ее обозначают 1 и false ("ложь") - обозначают 0. Поэтому переменную isAgreeможно прочесть как "дано согласие?" и так как значение равно 0, очевидно, что согласие не дано. 

Для обозначения переменных используется нотация camelCase - где каждое новое слово в названии переменной пишется с заглавной буквы. Эта же нотация принята в других языках - PHP, Java, C#. А вот в python используется нижнее подчеркивание (Underscore notation): camel_case.

Название переменной должно отражать данные, которые в ней хранятся. Так будет проще писать код и проводить отладку. 

В программировании корпоративных систем встречаются такие названия:

let welcomeMessageAuthUser;
let activeNewsInCategory;
let usersWithProfilesAndPosts;

Надо стремиться к тому, что бы имя переменной было более кратким, но в то же время отражало назначение переменной. В компаниях существуют свои соглашения об именовании переменных, которые могут отличаться от общепринятых.

Часто встречается и такой код:

const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const IS_ADMIN = 0;
const PI = 3.14;

Заглавными буквами объявляются константы, которые планируется много раз использовать в коде  или они известны заранее. То есть значение константы не зависит от выполнения функций или внешних изменений.

С другой стороны, если значение получено извне, то используется camelCase:

let num = 10;
const numFromUser = num;

Наверное, уже появился вопрос, а можно ли назвать переменную, к примеру, 1a? К нашей радости или сожалению - нет. Есть два правила, которые описывают, какое имя переменной допустимо:

  1. Имя переменной содержит только буквы, цифры или символы $ и _.
  2. Первый символ не должен быть цифрой.

Поэтому, имена ниже недопустимы:

let 1222hello; 

let my-house;

А вот имена ниже допустимы:

let $ = 1; 
let _ = 2;

И последняя важная новость - переменные зависят от регистра, т.е. let hello и let HeLlO - разные переменные.
